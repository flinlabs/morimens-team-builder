## 1. Data layer ✅

The reference data the algorithm reads. Regenerated by `npm run sync` (gitignored; see §6).

- ✅ **SKeyDB sync** — `scripts/sync-skeydb.mjs`. Fetches catalogs + per-record text, hydrates descriptions, retags Mythic wheels, detects character-bonus posses (15), derives `isDivineRealm`/`isLemurian`, merges annotations.
- ✅ **`db/awakeners.json`** — 57 awakeners, full hydrated skills/talents/enlightens, Divine (4) + Lemurian (5) flags, annotation merged, `annotationPending: false` for all 57.
- ✅ **`db/wheels.json`** — 134 wheels, 15 Mythic retagged, all with description text.
- ✅ **`db/covenants.json`** — 21 covenants, all with setEffects.
- ✅ **`db/posses.json`** — 51 posses, 15 character-bonus posses flagged (`hasCharacterBonus` / `characterBonusFor`).
- ✅ **`db/dzones.json`** — 62 D-Tide seasons + enemy characteristics (scheduling metadata only).
- ✅ **`db/sync-report.json` / `db/sync-snapshot.json`** — human-readable diff + searchTags snapshot for next-run diffing.
- ✅ **`annotations/awakeners.json`** — all 57 hand-annotations (roles, tiers, viability floors, breakpoints, anchor/recommended posses, pairings, key skills/talents, arc notes).
- ✅ **`app/admin/annotations/meta-teams.json`** — 34 verified, realm-legal reference comps.
- 🅿️ **Per-covenant-piece mainstat allocation table** — deep gear tuning (Mythag Covenants doc). Not needed for team composition; optional enrichment.

## 2. Knowledge layer ✅

Curated game-rules reference, queryable by the algorithm and usable as prompt context.

- ✅ **Arc 1 / Arc 2 ruleset** — `lib/arc-rules.ts`. Per-arc mechanic flags, rules-change reference, per-unit kit-shift modifiers. Wired into `viability.ts` scoring and the arc-aware Lemurian gate.
- ✅ **Status / keyword glossary** — `lib/status-glossary.ts`. 44 entries (combat statuses, resources, realm mechanics, card keywords) linked to the role/tag taxonomy.
- ✅ **Realm mechanics** — encoded in `filter.ts` (realm legality, mixing penalty, realm bonuses) + the glossary (tentacle stances, Ultra Space/Round, Devour, Crimson Furnace, etc.).
- 🅿️ **Boss / matchup knowledge** — deferred to a future opt-in mode (player selects the boss). Beginners get strong generic D-Tide teams without it. `dzones.json` only carries arc + realm, so this requires curating the Boss Compendium.

## 3. Algorithm layer (deterministic core) ✅

The engine. Selects teams and assigns gear with no AI in the loop.

- ✅ **Roster model + helpers** — `lib/roster.ts`. Owned awakeners/wheels/covenants/posses, enlighten/skill/talent levels, dual-SSR, arc + keeper settings, localStorage persistence.
- ✅ **DB loaders + read helpers** — `lib/db.ts`. Awakeners/wheels/covenants/posses/dzones, meta-teams, plus by-type/divine/lemurian/by-owner/by-realm/character-bonus helpers.
- ✅ **Viability scoring** — `lib/viability.ts`. Enlighten/skill/talent/wheel subscores → tiered score; arc-aware (Soulforge weight + per-unit arc kit-shift).
- ✅ **Realm legality + bonuses/penalties** — `lib/filter.ts`. ≤2 distinct realms (Chaos counts), mixing penalty, realm-bonus math (Chaos exemptions scoped correctly).
- ✅ **Role coverage** — `lib/filter.ts`. Maps team to covered roles + gaps.
- ✅ **Synergy scoring + requiresCondition** — `lib/filter.ts`. Pairing/synergy bonus; GMurphy Lemurian gate, arc-aware (Astral Reign only).
- ✅ **Candidate generation** — `generateCandidateTeams` (pinning, preferred realm, min tier, combinations, realm + coverage filters).
- ✅ **D-Tide 5-team solver** — `solveDtide` (greedy fill, no unit reuse).
- ✅ **Wheel / posse / covenant assignment** — `lib/assign.ts`. Owned-inventory wheel assignment with physical-item + dual-SSR constraints and FALLBACK targets; covenant recommendation; posse ranking (anchor/strong/situational).
- ✅ **TeamRecommendation assembler** — `lib/assign.ts`. `buildTeamRecommendation` + `buildDtideRecommendation` (shared `usedWheelIds` across 5 teams). Populates the `TeamRecommendation`/`CharacterAssignment` types.

> The deterministic algorithm is complete end-to-end: roster → ranked, geared team recommendations.

## 4. Generation entry point ✅ (AI narration deferred)

The algorithm is wired to a request. Deterministic and AI-free.

- ✅ **Entry point** — `app/api/generate/route.ts` (POST) over `lib/generate.ts` (`generateTeams`). The service loads the DB, runs the pipeline, returns ranked, geared `TeamRecommendation`s + meta.
- ✅ **Mode handling** — `single` (top-ranked alternatives, geared independently) vs `dtide` (5-team lineup, shared wheel pool). Boss-targeted mode is 🅿️ deferred.
- ✅ **Options/validation** — single-mode `FilterOptions` (pinned, preferred realm, min tier, max results); route validates JSON, mode, and roster shape (400s), and returns a "run `npm run sync`" hint on a missing-DB 500. Empty/insufficient rosters return warnings, not crashes.
- ⬜ **Optional AI narration pass** — thin layer that only *explains* the already-decided output. `ANTHROPIC_API_KEY` is set. Keep strictly post-hoc.
